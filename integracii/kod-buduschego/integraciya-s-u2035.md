---

1. Написать запрос в У2035 для получения данных авторизации для FTP У2035 и предоставить эти данные во Flow CRM. Это необходимо для получения/передачи части данных по заявкам.

2. Запросить в У2035 данные для авторизации запросов по API (для тестового стенда и прода) и предоставить их во Flow CRM. Это также необходимо для получения ещё одной части данных с У2035.

3. Во внешние источники следует добавить:

   1. внешние источники и их ключевые слова (порядок добавления не важен). Данные с У2035 и Odin будут сохраняться/отправляться в эти внешние источники. Эти внешние источники отображаются на странице каждой заявки или на странице программы/потока.

   2. дополнительные поля  и их ключевые слова (порядок добавления не важен). Доп данные заполнит ученик через свой личный кабинет. Они будут отображаются на странице каждой заявки в разделе «Личная информация».

   3. дополнительные документы.

      :::info 

      [Какие данные необходимо добавить? ](./programmy-i-potoki/trebuemye-dlya-kb-dop-polya-i-vneshnie-istochniki)

      :::

4. Завести шаблоны договоров для детей 18+ и младше 18 лет, если они у них разные, и сообщить во Flow CRM идентификаторы этих шаблонов. Каждой заявке, исходя из возраста, будет автоматически назначаться определенная группа шаблонов.

5. Завести отдельные шаблоны для оффлайн и онлайн формата обучения, также с учетом возрастной группы. И сообщить во Flow CRM идентификаторы этих шаблонов.

:::note 

Если в шаблонах для оффлайн программ будет использовано конкретное наименование организации, на площадке которой будет оказываться обучение, то организации необходимо предоставить во Flow CRM полный перечень сопоставления адреса проведения обучения и названия площадки.

:::

--

**Получив все эти данные, Flow CRM настроит подключение к FTP и API У2035.**

**--**

:::info 

**Как работает обмен данными с У2035**

Каждая заявка идентифицируется через **UntiId**. Данные обновляются двумя способами: с FTP приходят первичные данные (ФИО, СНИЛС), по API -- статусы и баллы с портала У2035. Сопоставление заявок происходит по UntiId и идентификатору курса.

Приказы на зачисление на У2035 оформляются **вручную** на портале -- Flow их не отправляет.

**Жизненный цикл модуля**

После прохождения промежуточной аттестации (ПА) в Odin результат автоматически приходит во Flow. На его основе в полуавтоматическом режиме генерируется сертификат по шаблону организации (шаблон должен быть заранее согласован с У2035 и загружен во Flow). Заявке присваиваются технические теги-статусы по итогам модуля и генерации сертификата. После генерации сертификата вручную запускается отправка данных об успешном завершении модуля в У2035. Зачисление на следующий модуль также производится вручную на портале У2035.

**М4 и итоговая аттестация**

Завершение М4 и ИА -- единый пакет данных. Сначала ученик завершает М4 в Odin, получает допуск к ИА, проходит ИА в Odin -- и только после этого Flow отправляет все данные в У2035 и генерирует сертификат за М4.

**ИТиСИ**

Допуск к ИТиСИ и подтверждение согласия ученика происходят в Odin. Само тестирование -- на платформе У2035.

:::
