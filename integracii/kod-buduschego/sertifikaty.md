:::info 

Сертификаты генерируются  в полуавтоматическом режиме после того, как Odin передаёт результат промежуточной аттестации по модулю. Для этого провайдер заранее предоставляет в поддержку Flow CRM **шаблон сертификата**, одобренный У2035.

:::

После генерации сертификат сохраняется в заявку и становится доступен ученику в его личном кабинете.

Сертификаты предусмотрены по каждому модулю:

-  Сертификат М1 -- по итогам 1 модуля

-  Сертификат М2 -- по итогам 2 модуля

-  Сертификат М3 -- по итогам 3 модуля

-  Сертификат М4 -- по итогам 4 модуля

:::quote 

**Важно:** завершение М4 и итоговой аттестации -- это единый пакет данных для У2035. Flow отправляет их в У2035 только после получения всех результатов из Odin.

:::

## Технические теги заявок

Заявкам по проекту КБ автоматически присваиваются **технические теги** -- они отражают статус прохождения модулей и итоговой аттестации. Эти теги нельзя удалить вручную, они проставляются системой автоматически на основе данных из Odin.

## Как устроена генерация сертификатов

Сертификаты в проекте КБ выдаются по итогам каждого из четырёх модулей. 

Генерация происходит в полуавтоматическом режиме: система формирует сертификаты по шаблону организации, как только получает из Odin подтверждение успешного прохождения модуля.

**Что нужно подготовить заранее:**

-  Предоставить во Flow CRM **шаблон сертификата**, одобренный У2035. У каждой организации свой шаблон.

-  В карточке организации во вкладке «Дополнительные данные» добавить четыре дополнительных документа с названиями: «Сертификат М1», «Сертификат М2», «Сертификат М3», «Сертификат М4». После генерации сертификаты будут сохраняться именно туда. [Подробнее](./programmy-i-potoki/trebuemye-dlya-kb-dop-polya-i-vneshnie-istochniki#дополнительные-документы)

### Генерация со страницы потока

На странице потока (в меню «...») доступна кнопка **«Сгенерировать сертификат»**. Она появляется только у программ с признаком проекта «Код Будущего».

При нажатии открывается окно с выбором **даты сертификата** -- эта дата будет указана в документе. После подтверждения система автоматически генерирует сертификаты сразу для всех заявок потока, у которых есть тег «Успешно М№».

**Сценарий 1 -- все сертификаты сгенерированы успешно:**

-  Появляется сообщение об успешной генерации

-  Каждой заявке присваивается тег «Сертификат М№»

-  Сертификат сохраняется в доп. документы заявки и становится доступен ученику в ЛК

**Сценарий 2 -- часть сертификатов не сгенерирована:**

-  Система сообщает, для скольких заявок сертификат сгенерирован, а для скольких -- нет

-  Заявкам с ошибкой присваивается тег «Ошибка генерации сертификата М№: \[описание ошибки\]»

-  После устранения ошибок в заявках можно нажать кнопку повторно -- система сгенерирует сертификаты только для тех заявок, у которых ещё нет тега «Сертификат М№»

:::quote 

Заявки, у которых уже есть тег «Сертификат М№», при повторной генерации пропускаются -- их сертификаты не перезаписываются.

:::

## Генерация из карточки заявки

В карточке отдельной заявки в разделе «Действия с документами» доступна кнопка **«Сгенерировать/перегенерировать сертификат»**. Она отображается только у заявок с тегом «Успешно М№».

Кнопка позволяет:

-  **Сгенерировать** сертификат, если у заявки его ещё нет

-  **Перегенерировать** -- если сертификат уже был выпущен, но его нужно обновить (например, при исправлении данных)

При перегенерации старый сертификат в доп. документах заменяется новым.

## Теги по сертификатам

| **СИТУАЦИЯ**                | **ТЕГ**                         |
|-----------------------------|---------------------------------|
| **Сертификат сгенерирован** | Сертификат М1М2М3М4             |
| **Ошибка генерации**        | Ошибка генерации сертификата М№ |

Теги «Сертификат М№» и «Ошибка генерации сертификата М№» взаимоисключающие -- у заявки по каждому модулю может быть только один из них. При успешной повторной генерации тег ошибки заменяется тегом «Сертификат М№».

### Особенность М4 и итоговой аттестации

Завершение 4 модуля и итоговой аттестации -- **единый пакет данных** для У2035. Порядок такой:

1. Ученик завершает М4 в Odin и получает допуск к ИА

2. Проходит итоговую аттестацию в Odin

3. Только после получения всех данных из Odin Flow CRM отправляет их в У2035

4. Генерируются сертификаты за М4 

:::quote 

Сертификат М4 не генерируется до тех пор, пока ученик не прошёл итоговую аттестацию.

:::

**По М4 Flow передаёт на У2035 следующие данные:**

-  Дата прохождения Промежуточной аттестации (ПА) М4

-  Дата документа, подтверждающего ПА М4 (дата Сертификата)

-  Тип и номер документа, подтверждающего ПА М4 («Сертификат, №...»)

-  Дата прохождения Итоговой аттестации

-  Тип и номер распорядительного акта ИА ("Аттестационная ведомость", №…)

-  Дата распорядительного акта ИА (дата Аттестационной ведомости)

-  Название итогового проекта

-  Адрес места обучения (только для офлайн-формата)
