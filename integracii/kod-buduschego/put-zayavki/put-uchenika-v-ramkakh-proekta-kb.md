-  Ученик оставляет заявку на Госуслугах, проходит вступительные испытания

-  При успешном прохождении получает ссылку на вход в личный кабинет Flow CRM, включает уведомления в ЛК (Flow) (как только заявка с У2035 поступит во Flow)

-  Cистема автоматически определяет группу шаблонов для заявки по возрасту (до 18 / 18+) и формату программы (онлайн / офлайн). Образовательная организация может поменять программу у заявки и выбрать поток (Flow), **измененные данные должны быть полностью идентичны с данными в У2035**

-  Ученик в ЛК проверяет и дополняет свои персональные данные (данные свидетельства о рождении -- если младше 14 лет, данные паспорта -- если старше 14 лет), заполняет данные законного представителя (если нет 18 лет; для совершеннолетних эти поля заполняются нулевыми значениями автоматически) (Flow)

-  Ученик (или его законный представитель) подписывает договор, заявление на зачисление и согласие на обработку персональных данных на шаге «Оформление заявления на зачисление» (Flow)

-  Образовательная организация проверяет загруженные документы и либо отклоняет для исправления, либо принимает (Flow)

-  После одобрения документов ученик попадает на шаг «Отправка оригиналов документов» -- в зависимости от настроек организации можно принести бумажные документы лично, отправить почтой (введя в ЛК трек-номер письма) или бумажные документы не требуются совсем (Flow или вне Flow) (Если выбрано получение бумажных документов -- в заявке проставляется отметка о получении оригиналов (Flow))

-  В день старта потока при наличии одобренных документов на зачисление приказ генерируется автоматически (Flow) / добавляется вручную сотрудником.[ Подробнее о приказах ](./../../../obuchenie/prikaz/_index)

:::info 

Приказы из Flow в У2035 не передаются. Сотрудник организации самостоятельно вносит данные о приказах в ЛК У2035.

:::

-  В день старта обучения при наличии приказа на зачисление у ученика в ЛК появляется кнопка для входа в Odin и приходит письмо-приглашение. Если приказа ещё нет -- ученика можно добавить в Odin вручную по кнопке «Зарегистрировать в Odin» на странице заявки и параллельно продолжать сбор документов (Flow)

-  Ученик поочерёдно проходит 1, 2, 3, 4 модули в Odin. После прохождения каждого модуля результат автоматически приходит во Flow. В случае успешного прохождения заявке присваивается тег «Успешно М№» и в полуавтоматическом режиме генерируется сертификат по шаблону организации. В случае неуспешного прохождения присваивается тег «Неуспешно М№» (Odin -> Flow)

-  После успешного прохождения всех четырёх модулей ученик получает допуск к итоговой аттестации (ИА) и проходит её в Odin (Odin)

-  Завершение М4 и ИА -- единый пакет данных: только после получения всех результатов из Odin Flow генерирует сертификат за М4 и отправляет данные в У2035 (Flow -> У2035). 

-  В случае успешного завершения обучения заявке присваивается тег «Успешно ИА», во Flow оформляется приказ об отчислении с успешным завершением, баллы фиксируются (Flow)

-  Ученик в Odin подтверждает согласие на прохождение ИТиСИ (Итоговому тестированию и социологическому исследованию) и переходит по ссылке для прохождения тестирования на платформе У2035 (Odin -> У2035)
